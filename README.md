# Тестирование модуля "Комнаты" IoT-платформы Smart Home

Проект содержит артефакты функционального, UI/UX и интеграционного тестирования REST API системы управления умным домом в условиях отсутствия документации (реверс-инжиниринг контрактов через DevTools и Swagger).

---

## 🛠 Технологический стек & Методы
* **Postman** — создание и запуск коллекций интеграционных тестов (сценарии авторизации и каскадного удаления).
* **Swagger (OpenAPI 3.1)** — верификация контрактов API, анализ статус-кодов (200, 409, 422).
* **Chrome/Яндекс DevTools** — анализ сетевого трафика (`Network`), локализация ошибок бэкенда.
* **Тест-дизайн:** Анализ граничных значений (BVA), Эквивалентное разделение (EP), Таблицы принятия решений для комбинаторной валидации полей.

---

## 🏗 Интеграционное тестирование API & Тестовые данные
В рамках проекта был смещен фокус с изолированной проверки фронтенда на сквозную проверку бизнес-логики и связей сущностей: `Users` ➡️ `Roles` ➡️ `Rooms` ➡️ `Devices` ➡️ `Stats`.

### Реализованные сценарии в Postman:
1. **Авторизация (Happy Path):** `POST /api/auth/login` с валидными данными ➡️ Извлечение `access_token` ➡️ Передача токена в Headers для `GET /api/users/me`.
2. **Деструктивный тест каскадных ограничений (Поиск багов бэкенда):** 
   * `POST /api/rooms` (Данные: `{"name": "Кухня", "area": 15}`) ➡️ Сохранение `room_id`.
   * `POST /api/devices` (Данные: `{"name": "Датчик дыма", "room_id": <room_id>}`).
   * `DELETE /api/rooms/<room_id>` — проверка обработки связей базой данных. *Выявлен критический дефект бэкенда.*
3. **Валидация граничных периодов:** `POST /api/stats-manadgement/{device_id}/auto` с передачей аномального диапазона дат (из будущего / нулевой период) для проверки устойчивости бизнес-логики.

## 🐛 Ключевые зафиксированные дефекты

### Bug №1 [Backend] — Критическая ошибка сервера (Error 500) при передаче некорректных параметров пагинации
* **Severity:** Major | **Priority:** High
* **Шаги воспроизведения:** Отправить GET-запрос `GET /api/rooms?limit=invalid_text`.
* **Фактический результат:** Статус `500 Internal Server Error`. Бэкенд падает, не обрабатывая исключение типа данных.
* **Ожидаемый результат:** Статус `422 Unprocessable Entity` с валидным JSON-описанием ошибки.

### Bug №2 [Logic/UI] — Установка отрицательного значения площади комнаты через UI-степперы
* **Severity:** Major | **Priority:** High
* **Шаги воспроизведения:** В окне добавления комнаты кликнуть на нижний степпер (стрелочка "вниз") в поле "Площадь".
* **Фактический результат:** Значение по умолчанию (1) уменьшается до `-1`. При нажатии "Добавить" запрос уходит на сервер, комната с отрицательной площадью сохраняется.
* **Ожидаемый результат:** Нижний лимит жестко ограничен единицей (1). При достижении минимума кнопка "вниз" становится неактивной (disabled).
