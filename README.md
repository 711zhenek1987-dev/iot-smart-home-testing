# Функциональное и UI/UX тестирование модуля "Комнаты" (IoT-платформа Smart Home)

Этот проект демонстрирует навыки глубокого тест-дизайна, кроссбраузерного тестирования интерфейсов и исследовательского тестирования веб-форм в условиях отсутствия ТЗ.

## 📊 Стратегия тест-дизайна и декомпозиция формы
Для формы "Добавить комнату" была проведена полная декомпозиция элементов интерфейса. Спроектирована и выполнена матрица из **82 высокодетализированных проверок** в трёх браузерах (Chrome, Firefox, Яндекс.Браузер).

### Примененные техники:
* **Классы эквивалентности (EP):** Поля "Название" и "Описание" проверены на типы данных (кириллица, латиница, спецсимволы, цифры, пробелы).
* **Граничные значения (BVA):** 
  - Нащупаны лимиты текстовых строк (валидные диапазоны: 3–30 символов для Названия, до 100 символов для Описания).
  - Проверены экстремальные значения для поля "Площадь" (от 1 до 999 999).

## 🐛 Локализованные UI/UX дефекты (Баг-репорты)

### Баг №1: Установка отрицательного значения площади комнаты через UI-степперы
* **Severity:** Major | **Priority:** High
* **Шаги:** В окне добавления комнаты кликнуть 2 раза на нижнюю стрелочку "вниз" в поле "Площадь".
* **Фактический результат:** Значение уменьшается до `-1` и форма отправляется на сервер.
* **Ожидаемый результат:** Нижний лимит ограничен единицей (1), при достижении минимума кнопка "вниз" блокируется.

### Bug №2: Непоследовательное поведение индикации ошибок валидации
* **Severity:** Normal | **Priority:** Medium
* **Шаги:** Очистить значение по умолчанию в поле "Площадь", поле "Название комнаты" оставить пустым. Нажать "Добавить".
* **Фактический результат:** Ошибка всплывает только под полем "Название". Под полем "Площадь" индикации нет.
* **Ожидаемый результат:** Сообщения об ошибках отображаются синхронно под всеми незаполненными обязательными полями.

### Bug №3: Риск потери данных пользователя при работе с модальным окном
* **Severity:** Normal | **Priority:** Low
* **Шаги:** Заполнить форму "Добавить комнату", кликнуть мышкой вне области модального окна.
* **Фактический результат:** Окно закрывается, все введенные пользователем данные безвозвратно теряются.
* **Ожидаемый результат:** Окно не должно закрываться по клику на overlay, либо должно выводиться предупреждение.
